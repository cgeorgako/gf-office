# Εργαλεία AutoLISP για Διαγράμματα Γεωμετρικών Μεταβολών (ΔΓΜ) — ProgeCAD

Ομάδα εντολών AutoLISP για τη σύνταξη Διαγραμμάτων Γεωμετρικών Μεταβολών
σύμφωνα με τις τεχνικές προδιαγραφές ηλεκτρονικής υποβολής του Ελληνικού
Κτηματολογίου (ν.4602/2019). Η δομή των layers και των πινάκων βασίστηκε σε
πραγματικά, αποδεκτά DXF υποβολών του γραφείου.

## Αρχεία

| Αρχείο | Περιγραφή |
|---|---|
| `DGM.lsp` | **Δοκιμάστε πρώτα αυτό.** Κωδικοποίηση UTF-8 με BOM — τη διαβάζουν σωστά τα σύγχρονα ProgeCAD (2021 και μετά / IntelliCAD 10+), ανεξάρτητα από τη γλώσσα των Windows. |
| `DGM-ansi.lsp` | Το ίδιο σε ANSI/Windows-1253 — **αν με το `DGM.lsp` τα ελληνικά βγαίνουν αλαμπουρνέζικα**, φορτώστε αυτό (παλαιότερες εκδόσεις ProgeCAD σε ελληνικά Windows). |
| `DGM.mnu`, `DGM-ansi.mnu` | Πτυσσόμενο μενού «ΔΓΜ» (Υπόβαθρο → Γεωμετρία/Πράξεις → Πίνακες → Έλεγχος → Εξαγωγή), στις ίδιες δύο κωδικοποιήσεις. Φόρτωση με την εντολή `MENULOAD`. |
| `src/DGM.utf8.lsp`, `src/DGM.utf8.mnu` | Πηγαία αρχεία σε σκέτο UTF-8, για ανάγνωση/επεξεργασία στο GitHub. Μετά από αλλαγές, ξαναδημιουργήστε τα αρχεία διανομής (βλ. παρακάτω). |

### Αν τα ελληνικά δεν διαβάζονται

1. **Στη γραμμή εντολών/μενού του ProgeCAD**: φορτώσατε τη «λάθος» έκδοση για
   την εγκατάστασή σας — δοκιμάστε το άλλο αρχείο (`DGM.lsp` ↔ `DGM-ansi.lsp`,
   και αντίστοιχα το μενού). Φροντίστε πρώτα να ξεφορτώσετε/κλείσετε το σχέδιο
   ή επανεκκινήστε το ProgeCAD ώστε να μην είναι φορτωμένα και τα δύο.
2. **Στα κείμενα που μπαίνουν στο σχέδιο** (πίνακες, ΚΑΕΚ): το text style δεν
   έχει ελληνική γραμματοσειρά — ορίστε στο τρέχον style μια TrueType όπως
   Arial (εντολή `STYLE`), ή SHX με ελληνικά (π.χ. greek.shx).
3. **Στο GitHub / σε editor**: ανοίξτε το `src/DGM.utf8.lsp` — τα αρχεία
   `*-ansi.*` φαίνονται πάντα αλαμπουρνέζικα σε UTF-8 προβολή· είναι
   φυσιολογικό.

> **Μην επεξεργάζεστε απευθείας τα αρχεία διανομής.** Αλλαγές κάντε στο
> `src/DGM.utf8.lsp` και ξαναφτιάξτε τα:
> `DGM.lsp` = UTF-8 **με BOM**, `DGM-ansi.lsp` = μετατροπή με
> `iconv -f UTF-8 -t CP1253` (ή Notepad → Save As → Encoding: ANSI, σε
> ελληνικά Windows).

## Εγκατάσταση στο ProgeCAD

1. Αντιγράψτε το `DGM.lsp` σε σταθερό φάκελο, π.χ. `C:\CAD\lisp\`.
2. Στο ProgeCAD: **Tools → Load Application** (εντολή `APPLOAD`) → επιλέξτε το `DGM.lsp`.
3. Για αυτόματη φόρτωση σε κάθε σχέδιο, προσθέστε το στη λίστα **Startup** του APPLOAD.
4. (Προαιρετικά) Φορτώστε και το μενού: εντολή `MENULOAD` → επιλέξτε το `DGM.mnu`.
   Θα εμφανιστεί πτυσσόμενο μενού **ΔΓΜ** με τις εντολές ομαδοποιημένες κατά
   βήμα εργασίας.
5. Πληκτρολογήστε `DGMHELP` για τη λίστα εντολών.

**Γραμματοσειρά:** βεβαιωθείτε ότι το τρέχον text style υποστηρίζει ελληνικά
(π.χ. Arial ή greek.shx), αλλιώς τα κείμενα των πινάκων θα εμφανίζονται λάθος.

## Εντολές

| Εντολή | Λειτουργία |
|---|---|
| `DGML` | Δημιουργεί όλα τα τυποποιημένα layers ΔΓΜ (PST_KAEK, DGM_PROP_FINAL, TOPO_PROP, AREA_A/D, DBOUND_*, κ.λπ.) με τα χρώματα των υποδειγμάτων. |
| `DGMK` | Αρίθμηση κορυφών polyline: βάζει POINT + κύκλο + αριθμό σε layer `σημείο_<layer>` (όπως στα υποδείγματα). Ο μετρητής συνεχίζει από την προηγούμενη εντολή. |
| `DGMP` | Πίνακας συντεταγμένων κορυφών (Σημείο, Χ, Y, Αποστάσεις) με τυποποιημένους τίτλους: αρχικό/τελικό γεωτεμάχιο, αποκοπτόμενο/διεκδικούμενο τμήμα, κορυφές ιδιοκτησίας, ή ελεύθερος τίτλος. Παίρνει τους αριθμούς κορυφών από τα σημάδια του `DGMK`. Σπάει αυτόματα σε πολλαπλές ομάδες στηλών όταν οι κορυφές είναι πολλές. |
| `DGME` | «ΠΙΝΑΚΑΣ ΑΡΧΙΚΩΝ ΚΑΙ ΤΕΛΙΚΩΝ ΕΜΒΑΔΩΝ ΕΠΗΡΕΑΖΟΜΕΝΩΝ ΓΕΩΤΕΜΑΧΙΩΝ» — τα εμβαδά υπολογίζονται αυτόματα με επιλογή polyline ή πληκτρολογούνται. |
| `DGMT` | «ΠΙΝΑΚΑΣ ΓΙΑ ΤΗ ΔΙΟΡΘΩΣΗ ΤΩΝ ΓΕΩΤΕΜΑΧΙΩΝ» (ΤΜΗΜΑ, ΕΜΒΑΔΟΝ, ΑΠΟ ΚΑΕΚ, ΣΕ ΚΑΕΚ). |
| `DGMA` | Χαρακτηρίζει τμήμα ως αποκοπτόμενο (AREA_A) ή διεκδικούμενο (AREA_D), μεταφέρει την polyline στο σωστό layer και βάζει ετικέτα «ΤΜΗΜΑ Α1/Δ1». |
| `DGMKAEK` | Τοποθέτηση κειμένου ΚΑΕΚ σε PST_KAEK (επηρεαζόμενο) ή PST (όμορο), με έλεγχο ότι έχει 12 χαρακτήρες. |
| `DGMSPLIT` | **Κατάτμηση**: κόβει κλειστό πολύγωνο με γραμμή κοπής (polyline που το διαπερνά) σε δύο νέα κλειστά πολύγωνα, με αναφορά εμβαδών. |
| `DGMUNION` | **Συνένωση**: ενώνει 2+ όμορα κλειστά πολύγωνα σε ένα, αφαιρώντας τα κοινά όρια. Προϋπόθεση: ταυτισμένες κορυφές στα κοινά όρια (εντός ανοχής). |
| `DGMCUT` | **Αποκοπή**: αφαιρεί τμήμα (π.χ. αποκοπτόμενο Α1) από γεωτεμάχιο και παράγει το υπόλοιπο ως νέο κλειστό πολύγωνο, με σύγκριση εμβαδού. |
| `DGMHASH` | Υπολογίζει το **SHA512 hash** αρχείου DXF (μέσω `certutil` των Windows) και προαιρετικά τοποθετεί το hash ως κείμενο στο σχέδιο, για αναγραφή στο PDF. Εναλλακτικά εκτός CAD: `tools/dxf-hash.cmd`. |
| `DGMKHD` | Τοποθέτηση του ΚΗΔ (Κωδικού Ηλεκτρονικού Διαγράμματος που εκδίδει η εφαρμογή υποβολής) ως κείμενο στο σχέδιο. |
| `DGMC` | Έλεγχοι πριν την υποβολή: ύπαρξη υποχρεωτικών layers, κλειστές LWPolylines, απαγορευμένοι τύποι οντοτήτων (LINE/ARC/SPLINE στα όρια), τόξα σε polylines, διπλές κορυφές, κείμενο ΚΑΕΚ μέσα σε κάθε πολύγωνο PST_KAEK, ταύτιση κορυφών DGM_PROP_FINAL με τα όρια PST_KAEK (με ανοχή), σύγκριση αθροισμάτων εμβαδών. Τα προβλήματα επισημαίνονται με κύκλους στο layer `DGM_CHECK`. |
| `DGMHELP` | Λίστα εντολών. |

## Προτεινόμενη ροή εργασίας

1. `DGML` — δημιουργία layers.
2. Σχεδίαση/τακτοποίηση: όρια κτηματολογίου σε `PST_KAEK` (+ ΚΑΕΚ με `DGMKAEK`), όμορα σε `PST`, ιδιοκτησία κατά τίτλους σε `TOPO_PROP`, τελική γεωμετρία σε `DGM_PROP_FINAL` — όλα **κλειστές** LWPolylines.
3. `DGMA` — χαρακτηρισμός αποκοπτόμενων/διεκδικούμενων τμημάτων.
4. `DGMK` — αρίθμηση κορυφών κάθε πολυγώνου (TOPO_PROP → PST_KAEK → DGM_PROP_FINAL, ενιαία αρίθμηση).
5. `DGMP` — πίνακες συντεταγμένων για αρχικά γεωτεμάχια, τμήματα και τελικά γεωτεμάχια.
6. `DGME`, `DGMT` — πίνακες εμβαδών και μεταβολών.
7. `DGMC` — τελικός έλεγχος. Διορθώστε ό,τι επισημανθεί, σβήστε το layer `DGM_CHECK`.
8. Εξαγωγή DXF + ψηφιακά υπογεγραμμένο PDF με ίδιο όνομα αρχείου, υποβολή στην εφαρμογή του Κτηματολογίου.

## Χωρικές πράξεις — πώς δουλεύουν

Οι `DGMUNION`/`DGMCUT` βασίζονται σε «ακύρωση» των κοινών πλευρών: πλευρές που
εμφανίζονται και στα δύο πολύγωνα αφαιρούνται και οι υπόλοιπες αλυσιδώνονται σε
νέο κλειστό περίγραμμα. Πριν από την πράξη, κάθε πλευρά «σπάει» αυτόματα στις
κορυφές του άλλου πολυγώνου που πέφτουν πάνω της, ώστε να μη χρειάζεται τα δύο
πολύγωνα να έχουν ακριβώς τις ίδιες κορυφές στο κοινό όριο. Απαιτείται όμως το
κοινό όριο να **συμπίπτει γεωμετρικά** εντός της ανοχής (προεπιλογή 0.001 m) —
ό,τι δηλαδή απαιτούν έτσι κι αλλιώς οι προδιαγραφές και ελέγχει το `DGMC`.
Η `DGMSPLIT` απαιτεί η γραμμή κοπής να διαπερνά πλήρως το πολύγωνο (ακριβώς 2
σημεία τομής). Οπές/νησίδες δεν υποστηρίζονται.

Οι αλγόριθμοι έχουν επαληθευτεί πάνω σε πραγματικά ΔΓΜ του γραφείου: η αποκοπή
τμήματος αναπαράγει το τελικό πολύγωνο DGM_PROP_FINAL κορυφή προς κορυφή, και η
συνένωση 5 όμορων γεωτεμαχίων PST_KAEK (419 κορυφές) δίνει ενιαίο περίγραμμα με
ακριβές άθροισμα εμβαδών.

## Hash DXF και ΚΗΔ

Η ηλεκτρονική υποβολή συσχετίζει το PDF με το DXF. Η `DGMHASH` υπολογίζει το
αποτύπωμα **SHA512** του DXF ώστε να αναγραφεί στο διάγραμμα/PDF, και η
`DGMKHD` τοποθετεί τον ΚΗΔ που εκδίδει η εφαρμογή του Κτηματολογίου. Αν το
περιβάλλον LISP δεν υποστηρίζει συγχρονισμένη εκτέλεση (παλιές εκδόσεις
ProgeCAD), η εντολή ανοίγει παράθυρο cmd και περιμένει Enter· εναλλακτικά
χρησιμοποιήστε το `tools/dxf-hash.cmd` (σύρετε το DXF πάνω του).

## Σημειώσεις / περιορισμοί

- Οι συντεταγμένες πρέπει να είναι σε **ΕΓΣΑ '87** — τα εργαλεία γράφουν ό,τι
  υπάρχει στο σχέδιο, δεν κάνουν μετασχηματισμό.
- Ο έλεγχος `DGMC` είναι βοηθητικός προέλεγχος. Δεν υποκαθιστά τους ελέγχους
  της εφαρμογής ηλεκτρονικής υποβολής ούτε τις επίσημες προδιαγραφές:
  - [Οδηγία ηλεκτρονικής υποβολής ΤΔ & ΔΓΜ (ν.4602/19)](https://ktimatologio.gov.gr/Professionals/Content/%CE%9F%CE%94%CE%97%CE%93%CE%99%CE%91_%CE%A4%CE%9F%CE%A0%CE%9F%CE%93%CE%A1%CE%91%CE%A6%CE%99%CE%9A%CE%9F_%CE%94%CE%99%CE%91%CE%93%CE%A1%CE%91%CE%9C%CE%9C%CE%91_%CE%93%CE%95%CE%A9%CE%9C%CE%95%CE%A4%CE%A1%CE%99%CE%9A%CE%A9%CE%9D_%CE%9C%CE%95%CE%A4%CE%91%CE%92%CE%9F%CE%9B%CE%A9%CE%9D_%CE%9D4602_19.pdf)
  - [Τεύχος τεχνικών προδιαγραφών ψηφιακών αρχείων διαγραμμάτων](https://cdn.ktimatologio.hast.gr/EK_PRODIAGRAFES_PSIFIAKON_ARXEION_DIAGRAMMATON_v2_f13a300ccb.pdf)
- Οι πίνακες σχεδιάζονται με γραμμές και κείμενα (όχι ACAD_TABLE) στο layer
  `pinakas_sintetagmenon`, για πλήρη συμβατότητα ProgeCAD/IntelliCAD.
- Δοκιμασμένη στόχευση: ProgeCAD Professional (IntelliCAD). Δουλεύει και σε
  AutoCAD/BricsCAD/ZWCAD.
